# Incident: note rewrite-body で本文表現と関連リンクが崩れた

**日付**: 2026-04-03
**影響**: 公開済み note 記事 `nc3330321b750` の本文に `。。` 修正を入れる過程で、`##` と `**` の露出、関連リンク消失、不要空白行が発生した
**深刻度**: high

## 事象

公開済み記事「教育費と投資の優先順位が決められないときの整理」の本文修正時に、以下の不具合が連続発生した。

1. `その迷いを整理します。。` を直すために `rewrite-body` を実行したが、カード既存判定で本文再投入がスキップされ、修正が反映されなかった
2. スキップを止めて再実行すると、markdown 見出し `##` が公開面にそのまま表示された
3. 原稿内に残っていた壊れた `**` も公開面に露出した
4. 関連リンクの URL 2本が変換時に落ち、「あわせて読みたい」だけが残った
5. 空行が `<p><br></p>` に変換され、本文冒頭に不要な大きい空白が入った

## 直接原因

### 1. `rewrite-body` が生 markdown を打鍵していた

`note/ops.py` の `rewrite_body()` が `_input_body_text()` を使って本文を直接入力していたため、`##` 見出しや `**` が note 用 HTML に変換されず、そのまま公開面に出た。

### 2. 本文変換が「見出し + URL」の混在ブロックを扱えていなかった

`note/convert.py` の `_split_body_into_blocks()` は、空行区切りブロック単位で HTML 変換していた。このため以下のようなブロック:

```text
あわせて読みたい
https://note.com/...
https://note.com/...
```

を 1つの html ブロックとして扱い、URL 行を card として残せず、関連リンクを落としていた。

### 3. 空行を明示的な空段落へ変換していた

`note/convert.py` の `_split_body_for_note()` は空行を毎回 `<p><br></p>` に変換していた。note 側は通常の段落余白を持つため、これが公開面で不要に大きい空白として見えた。

## 間接原因

1. `rewrite-body` が新規投稿フローと別経路になっており、同じ変換処理を共有していなかった
2. 回帰確認が「文字修正」だけに寄っていて、見出し・太字・関連リンク・空行を含む実記事での検証が不足していた
3. 「公開面の HTML に期待要素が残っているか」の確認を最初から機械的に入れていなかった

## 暫定対応

- [x] 原稿 `note_articles/note_add_41_教育費と投資の優先順位が決められないときの整理.md` の壊れた `**` を除去
- [x] `rewrite-body` のカード既存スキップを本文修正用途で無効化
- [x] 公開記事 `nc3330321b750` を複数回再反映し、最終的に本文・見出し・関連リンクを正常化

## 恒久対応

### 実装修正

1. `note/cli.py`
   - `rewrite-body` で `skip_if_cards_existing=False` を指定し、本文修正時は再投入を必ず実行
2. `note/ops.py`
   - `rewrite_body()` を `_split_body_into_blocks()` + `_insert_body_blocks()` 経由に変更し、生 markdown 打鍵を廃止
3. `note/convert.py`
   - URL 行と通常本文が混在していても順序を保って html / card に分割するよう修正
   - 空行を `<p><br></p>` に変換しないよう修正

### 実記事確認

公開記事 `nc3330321b750` について以下を確認済み:

- `その迷いを整理します。。` がない
- `## 迷うのは` がない
- `その迷いを整理します**` がない
- 関連リンク `n071fff295d4a` と `nc053ef2751dd` がある
- 不要な `<p><br></p>` がない
