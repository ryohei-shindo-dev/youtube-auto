# Incident: note記事10本でリード文の句点が二重表示（。。）

**日付**: 2026-03-26
**影響**: 公開済みnote記事10本（AI整理シリーズ2本 + 追加記事8本）でリード文が「。。」と二重句点で表示
**深刻度**: low（表示上の問題のみ、内容の正確性には影響なし）

## 事象

ユーザーが note記事 `n1b93028cff20`（老後資金×AIで不安の中身を分ける方法）を確認したところ、冒頭リード文に「AIで分けて整理します。。」と句点が二重に表示されていた。

調査の結果、同じ問題が合計10本の記事で発生していた。

2つのパターンが存在：
- パターン1（7本）: `整理します。**。` — 太字閉じの後に余分な句点
- パターン2（2本）: `**。` が単独行に分離 — 太字の開きと閉じが改行で分断
- ai_08（1本）: パターン1と同じ

noteエディタは `**` をMarkdownとして解釈するため、結果的に「。。」と二重句点に見えていた。

## 直接原因

Claude（Haiku）が太字リード文を生成する際、`**` の閉じ位置を句点の後に配置し、さらに太字外にも句点を付けていた。

```markdown
# 正しい生成結果
AIで分けて整理します。**

# 実際の生成結果（パターン1）
AIで分けて整理します。**。

# 実際の生成結果（パターン2）
**相場が下がっている。追加で買えばチャンスかもしれない。

でも、もっと下がるかもしれない。その迷いを整理します。

**。
```

## 間接原因

- `note_gen.py` の `_save_article()` に生成結果のマークダウン構文チェック・後処理がなかった
- プロンプトで「太字で2文まで」と指示しているが、Haikuが `**` マーカーの位置を正確に扱えていなかった
- 生成後に人間によるレビューを経ずに投稿していたため、表示上の不具合が見逃された

## 暫定対応

1. ローカルファイル10本を正規表現で一括修正
2. Playwrightで公開済み10本の本文を全文再投入（全10本成功）

## 恒久対応

`note_gen.py` の `_save_article()` に後処理を追加：

```python
# 太字閉じの後の余分な句点を除去（Claude が 。**。 と生成するバグ対策）
article = re.sub(r'。\*\*。', '。**', article)
# 単独行の **。 を除去（太字閉じが改行で分離した場合）
article = re.sub(r'\n\*\*。\s*\n', '\n', article)
```

## 再発防止策

| 間接原因 | 対策 |
|---|---|
| LLM生成結果のマークダウン構文に後処理がなかった | `_save_article()` に正規表現による自動修正を追加（実装済み） |
| Haikuが太字マーカーの位置を誤る傾向 | 後処理で吸収する方針（プロンプト修正よりも確実） |
