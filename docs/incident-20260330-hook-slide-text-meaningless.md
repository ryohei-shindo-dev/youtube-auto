# 障害報告: hook slide_text が意味不明 + data で数字分断（2026-03-30）

## 事象

3/30 19:00投稿の YouTube Shorts（No.207「勉強するほどブレる人へ」）で2つの表示問題:

1. **hook スライド**: 「10年、判断が増えた」→ 意味不明（「勉強したのに」が欠落）
2. **data スライド**: 「10年で勝率9 / 5%以上」→ 「95%」が改行で分断

## 影響

- 視聴者に意味が伝わらないスライドが2枚表示された
- 削除・再投稿は不要（ユーザー判断）

## 原因

### 問題1: hook slide_text が意味不明

- **根本原因**: Claude API が生成した slide_text をそのまま使用していた
- ナレーション（text）は「10年。勉強したのに、判断が増えた。」で意味が通るが、Claude が slide_text を「10年、判断が増えた」と生成し、「勉強したのに」を省略
- empathy/resolve/closing の slide_text はコード側で自動生成しているが、hook だけは Claude 生成のまま通していた

### 問題2: data の数字分断

- **根本原因**: 数字切断修正（bf34387 / 2026-03-16 17:58）の前にスライドが生成されていた
- スライドファイル日付: 2026-03-16 16:29 → 修正コミットの1時間半前
- 現在のコードでは「95%」がプレースホルダ保護されるため再発しない

## 対応

### 問題1（修正）

hook の slide_text を Claude 生成に頼らず、ナレーション text から `_safe_truncate_slide_text(text, max_len=10)` で自動生成する方式に変更。

- コミット: 8761105
- 変更箇所: `script_gen.py` `_postprocess_script` 内の hook 処理ブロック
- 修正後の結果: 「10年。勉強したのに、判断が増えた」→「10年。勉強したのに」（意味が通る）

### 問題2（修正済み）

- bf34387（2026-03-16）で `_wrap_text_lines` に数字+単位のプレースホルダ保護を追加済み
- 現在のコードでは「95%以上」→ 保護対象として分断されない
- 追加対応不要

## 再発防止

- hook の slide_text は今後 Claude に生成させず、ナレーションから自動生成
- 既存キュー内の動画は生成済みのため、この修正は新規生成分から適用
