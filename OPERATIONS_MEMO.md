## Temporary Priorities
- 当面のテーマ設計は `入口 / 中核 / 土台` の3層で考える。
- `入口テーマ`: `含み損`、`暴落`、`売りたい`。新規視聴者の初速を取りやすい。
- `中核テーマ`: `比較疲れ`、`目移り`、`正解不安`。このチャンネルの本丸として優先する。
- `土台テーマ`: `退屈`、`何もしないことの意味`、`静かな継続`。思想面の補強に使う。
- 当面優先する構造は `比較 / 目移り / 焦り -> 自分の軸の再確認 -> 安心`。
- `恐怖 -> 歴史データ -> 安心` は主軸ではなく入口テーマとして使う。
- 横展開候補として `比較疲れ`、`目移り`、`正解不安` 系の派生テーマを優先する。
- マルチプラットフォームでは `TikTok` と `Instagram Reels` の強化を優先候補とする。
- 未確定の改善案は、質問型 CTA、ループ再生対応、シリーズ化。

## Channel Roles
- `YouTube Shorts`: 新規接触・テーマ検証・勝ち筋発見の主戦場。
- `Instagram Reels`: YouTube内容の再接触面。雰囲気と世界観の補助導線。
- `X`: 告知よりも人間味の補完を優先する補助接点。信用回復を優先。
- `note`: Shortsで動いた感情を静かに整理する深さの媒体。
- `YouTube長尺`: Shortsで反応が強かったテーマの受け皿。量産はしない。

## Cross-Platform Selection Flow
- 基本フローは `Shorts → note → 長尺`。
- すべてのテーマを全媒体に横展開しない。
- `Shorts`: まず反応を見るための起点。
- `note`: Shortsで反応が良いテーマだけ送る。
- `長尺`: noteでも伸ばせる強テーマのうち、音声と間で深める価値があるものだけ候補にする。

## Experiment Log
- CTA 試行案:
  - `続けてますか`
  - `同じ人いますか`
  - `含み損でも続けていますか`
- ループ再生案:
  - closing に hook ワードを再度含めて自然ループを作る。
  - 例: `暴落` で始まり、closing 側にも `暴落` を含める。
- シリーズ化案:
  - `暴落の歴史 #1 / #2 / #3`
  - `ガチホ哲学 #1 / #2`
- 派生量産案:
  - 元動画の `hook + data` を維持して派生テーマを増やす。
- hook文言メモ:
  - `増えない人へ` より `お金が増えない人へ` のように、何が増えないのかを明示した方が刺さりやすい仮説。
  - `短い=強い` ではなく、`短くても意味が即立つ=強い` として扱う。
  - 意味が即立たない失敗例メモ:
    - `手数料がつらい` → 何の手数料か分からず弱い。
    - `爆益で悩む` → 状況が伝わらず、自分事化しにくい。
    - `売りたい人へ` → 何を売りたいのかが抜けている。
    - `3年目が一番つらい` → 何の3年目か不明で、別文脈に読まれやすい。
- 表記・音声メモ:
  - タイトルやテキストの `積立` は、今後は `積み立て` に統一する。
  - closing の `あなたもガチホ仲間..` 系は音声崩れが出ることがあるため、文言または音声処理の見直し候補として扱う。
- note運用インシデントメモ:
  - note記事のヘッダー画像差し替え時、予約投稿済みだった記事の一部が本投稿になった。
  - 当面対応: すでに本投稿された記事はそのまま維持し、再公開や取り下げは行わない。
  - 原因調査メモ: `note_image_replace.py` は編集後に `/publish/` 画面へ遷移し、予約状態の再確認や予約日時の再設定をせずに `予約投稿 / 投稿 / 公開` ボタンを押している。
  - 実装上の懸念箇所: `note_image_replace.py` は publish 画面へ遷移した後、汎用セレクタで `予約投稿 / 投稿 / 公開` ボタンをクリックしている。`note_publish.py` のような予約日時設定処理は入っていない。
  - 推定原因: note側の編集フローで予約状態が保持されない、または publish 画面の既定動作が即時公開側に寄っており、画像差し替えスクリプトがそのまま確定してしまった可能性が高い。
  - 再発防止: `note_image_replace.py` は既定で `下書き保存まで` に変更し、公開確定は `--finalize-publish` 指定時だけにした。
  - 追加記事投稿時の学び: `note` はタイトル・本文が画面上に見えていても、内部状態で未確定のままだと `タイトル、本文を入力してください` モーダルが出ることがある。
  - 対応: 追加記事用の `note_publish_additional.py` では、`公開に進む` 前に必ず `下書き保存 / 一時保存` を押してから publish フローへ進むようにした。
  - 結果: 追加5本のうち1本を即時公開、4本を 2026-03-15〜18 21:00 で予約投稿できた。
- note追加方針メモ:
  - 3月末までの追加本数は `5〜7本` を目安にし、無理に増やさない。
  - 比率は `中核テーマ 7割 / 土台テーマ 3割` を優先し、入口テーマの追加は少なめでよい。
  - 足すべきなのは `怖いときの記事` より `揺れたときに戻る記事`。
  - note向きの中心は `比較疲れ`、`目移り`、`正解不安`、`退屈`、`何もしないことの意味`。
  - 3月末までの優先候補:
    - `オルカンでいいのか揺れる夜`
    - `S&P500が遅く見えるとき`
    - `他人の爆益を見た夜に、軸が揺れる`
    - `何もしないことが不安になる日`
    - `正しいのに退屈で続かない`
- 検証前の仮説と、検証後に採用しなかった案もここに残してよい。
