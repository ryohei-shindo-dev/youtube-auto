## 相談の目的
YouTube Shorts自動生成パイプラインのClaude APIコスト削減施策を7つ実装した。施策の妥当性確認と、次にやるべきことのアドバイスがほしい。

## 実装した7施策

### 施策A: 事前枯渇チェック（dedupe_check.check_exhaustion）
- **何をするか**: APIを叩く前に、そのトピックで新しいhook×data組み合わせが出せるか判定
- **仕組み**: `_TOPIC_HOOK_MAP` でトピック→生成されやすいhookパターンを推定。hook類似度≥80% かつ data類似度≥50% なら「ブロック済み」。全組み合わせがブロックならAPIスキップ
- **閾値**: 枯渇度80%以上→スキップ、50〜80%→リトライ1回に制限
- **効果**: 枯渇トピックへの無駄なAPI呼び出しを完全にブロック

### 施策E: hook/dataプール拡充（25個→60個）
- **変更内容**: DATA_POOL を各カテゴリ5→10個に拡充（暴落/長期/複利/心理/歴史/比較の6カテゴリ）
- **新設**: 「比較」カテゴリ（SNS焦り/レバレッジ/仮想通貨/個別株等のトピック向け、10個）
- **効果**: hook×dataの組み合わせ候補が大幅増加→重複による再生成が減少

### 施策C: プロンプトキャッシュ
- **何をするか**: テンプレート全体をsystemパラメータに分離し、`cache_control: {"type": "ephemeral"}` を付与
- **仕組み**: バッチ内で同じsystemプロンプトが再利用され、2回目以降はキャッシュヒットでインプットトークン節約
- **効果**: バッチ生成時のインプットトークンコスト削減

### 施策B: note記事モデル → Haiku
- **変更**: claude-sonnet → claude-haiku-4-5-20251001（約1/12のコスト）
- **理由**: note記事はShortsほど感情曲線の精度が不要。構成テンプレートが厳密なのでHaikuでも十分

### 施策: 1API3候補生成（generate_shorts_candidates）
- **何をするか**: 1回のAPI呼び出しで3パターンの台本をJSON配列で生成
- **仕組み**: Phase1で3候補をローカルスコアリング→最良を選択。全不合格時のみPhase2で1候補ずつリトライ（最大2回）
- **スコアリング基準**（candidate_ranker.py、最大12点、合格5点以上）:
  - タイトル数字（3点）: 具体的な数字があるか
  - hook強度（3点）: 強い痛みワードか
  - data具体性（2点）: 数字が含まれるか
  - 禁止表現（2点）: 禁止ワードがないか
  - トピック一致（2点）: タイトルがトピックに関連するか
- **効果**: 従来は最大3回のAPI呼び出し→ほとんど1回で済む

### 施策: Batch API（batch_api_gen.py）
- **何をするか**: Anthropic Messages Batch APIで一括送信（50%割引、非同期最大24時間）
- **4コマンド**: submit（送信）/ fetch（結果取得+動画生成）/ status（確認）/ run（一括実行）
- **フロー**: シートから未生成トピックを一括取得→Batch APIに投げる→完了待ち→結果取得→ポスト処理→動画パイプライン
- **効果**: 台本生成コスト50%割引（ただし即座には返ってこない）

### 施策D: 台本生成モデル → Haiku
- **変更**: claude-sonnet-4-6 → claude-haiku-4-5-20251001
- **テスト結果**: 同じ5トピックで比較
  - Sonnet平均: 5.8/12点
  - Haiku平均: 7.4/12点
  - Haikuの方がプロンプト指示に忠実（数字入りタイトル、短いhook等）
- **コスト**: 約1/5
- **切り戻し**: `.env` に `SCRIPT_MODEL=claude-sonnet-4-6` を追加するだけ

## 参考: 現在のシステム構成

```
Sheets(投稿管理) → script_gen(Haiku) → voice_gen(ElevenLabs) → slide_gen(Pillow)
  → video_gen(FFmpeg) → thumbnail_gen → subtitle_gen → note_gen(Haiku) → social_gen
  → done/{folder}/ にアーカイブ → Sheets に記録
  → auto_publish: 各プラットフォーム投稿
  → analytics_collect(毎晩) → analytics_analyze(毎週) → analytics_insights.json → script_genにフィードバック
```

台本生成の品質管理:
- candidate_ranker.py: 12点満点、5点以上で合格
- dedupe_check.py: タイトル類似度65%、hook類似度80%、hook×data組み合わせ判定
- 事前枯渇チェック: hook×data組み合わせの残余を事前判定
- 3候補一括生成 + ローカル選別 + フォールバックリトライ
