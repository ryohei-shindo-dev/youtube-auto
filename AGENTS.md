# AGENTS.md

## Purpose
- このリポジトリは、チャンネル「ガチホのモチベ」向けに、長期投資を続けるモチベーションを与える短尺動画・関連テキストを自動生成し、配信するためのもの。

## Product Rules
- このチャンネルは投資ノウハウ解説ではなく、長期投資を続けるための共感・思想・モチベーション提供を目的とする。
- トーンは「煽らない・助言しない・静かに寄り添う」。
- 個別株の推奨、買い時の断定、短期トレード推奨、誇張表現、サムネ詐欺は行わない。
- 生成物は、視聴後または読後に「長期投資を続けてよかった」と感じられる内容にする。
- 投資助言ではない旨の免責を維持する。

## Content Shape
- Shorts の基本構成は `hook -> empathy -> data -> resolve -> closing`。
- 最初に置く `hook` は、説明ではなく感情を止める痛みワードを優先する。
- 基本の感情曲線は `痛み/恐怖 -> 共感 -> データ -> 安心/希望`。
- 長い説明調より、短く意味が立つ表現を優先する。
- 視覚・音声の雰囲気は、落ち着いたブランドトーンを保つ。

## Long-Form Visual Rules
- 長尺動画は `音声主役 / 文字は補助` を徹底する。
- 長尺スライドは `1画面1メッセージ` を原則とし、文字量は `見出し + 1行` までを基本にする。
- 長尺では説明資料っぽい大きな黒パネルや授業用スライド感を避ける。
- 文字ボックスは常用せず、背景を見せる。余白を大きく取り、背景の空気感を主役にする。
- 背景は象徴映像の方向で選ぶが、強者・権力・高級車・札束のような威圧的モチーフは避ける。
- このチャンネルに合う象徴は `扉 / 灯り / 鍛える / 待つ / 耐える / 夜明け前 / 一人で考える人物 / 静かに進む`。
- 長尺の静止画演出は `scale→crop`（iMovie Ken Burns 方式）を使う。`zoompan` フィルターは揺れるので使わない。
- ズーム量は `100% → 103.5%` を固定値とし、中央基準で平行移動はほぼ入れない。
- ズーム方向はロールで固定する: 内面系（hook / empathy / why_painful / data）= zoom in、余韻系（interpret / action / closing）= zoom out。
- 動きの目的は「止まって見えない」ことであり、「動いていると分かる」ことではない。
- 動きは映画的に見せるための補助であり、派手なモーションにはしない。
- 参考にするのは `映像の静けさ / 背景の象徴性 / 文字の少なさ / 音声主役 / 間の使い方` まで。
- 持ち込まないのは `権威口調 / 支配感 / 強者ポエム / 重低音の圧 / 格言チャンネル化`。

## Source Of Truth
- 恒常ルールの一次資料はこの `AGENTS.md`。
- 戦略背景や企画意図の一次資料は `CHANNEL_STRATEGY.md`。
- 変動する運用情報の置き場は `OPERATIONS_MEMO.md`。
- `note_articles/*.md` はルールブックではなく、テーマ感・文体・補足説明の参考資料として扱う。
- 日付、在庫本数、投稿順、分析数値、仮説ランキングは変動情報として扱い、固定ルール化しない。

## Working Rules For Codex
- 方針変更時は、まず既存実装と `CHANNEL_STRATEGY.md` の整合を確認する。
- コンテンツ方針に関わる変更では、表現が投資助言や煽りに寄っていないかを最優先で確認する。
- 既存のブランドトーンを崩す変更は避ける。
- 変動しやすい運用メモは `AGENTS.md` ではなく別のメモファイルに分離する。

## What To Check Before Editing
- 台本生成や文言変更では、まず `CHANNEL_STRATEGY.md` と既存コードの両方を確認する。
- 記事文体や補足説明を調整する場合は `note_articles/*.md` を参考にして、静かで断定しすぎない語り口を保つ。
- 実装側の現行仕様を優先して確認する対象は `script_gen.py`、`social_gen.py`、`note_gen.py`、`thumbnail_gen.py`、`auto_publish.py`。

## Expression Checklist
- 説明より先に感情を置いているか。
- 不安や痛みに触れたあと、安心につながる構造になっているか。
- 数字やデータは安心材料として使われており、煽り材料になっていないか。
- 視聴者を責める文体になっていないか。
- 「今すぐ」「絶対」「必ず勝てる」のような過剰断定が入っていないか。
- 投資助言と受け取られうる表現が紛れ込んでいないか。

## Avoid Phrases
- `今が買い時`
- `この銘柄を買うべき`
- `絶対に儲かる`
- `誰でも勝てる`
- `これを知らないと損`
- `まだ買ってないのは危険`
- `短期で稼ぐ`
- `爆益確定`

## Prefer Phrases
- `不安になるのは自然`
- `あなただけではない`
- `過去のデータではこの傾向がある`
- `長期で見ると`
- `退場しないことが大事`
- `続ける理由を確認する`
- `投資助言ではありません`

## Style Rules
- ユーザー向け文言では `積立` ではなく `積み立て` を使う。見出し、タイトル、本文、SNS文面、字幕を含めて原則統一する。
- 固有名詞や制度名で漢字表記が不可避な場合を除き、生成文で `積立` をそのまま残さない。

## Temporary Info Handling
- 投稿本数、投稿日、再生数、順位、レビュー点数、当面の優先テーマは運用メモとして別管理する。
- 未確定の改善案や仮説は `AGENTS.md` に昇格させず、検証中メモとして扱う。
- 期間限定の施策は、恒常ルールに見える書き方を避ける。
- 変動情報を残す先は原則 `OPERATIONS_MEMO.md` とする。

## Out Of Scope
- Claude Code 固有のローカル権限設定や個人環境依存の設定はここに持ち込まない。
- 一時的な投稿計画、分析ログ、試行中の仮説は恒常ルールとして固定しない。
